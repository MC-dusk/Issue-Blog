# 解决 Scoop 网络问题

> Scoop 是个好东西，但是由于大量使用 Github 链接导致经常无法正常使用，使用镜像是个不错的解决方法。

## 配置镜像

配置 scoop 本身的镜像：

```shell
scoop config SCOOP_REPO https://ghgo.xyz/https://github.com/ScoopInstaller/Scoop
```

`ghgo.xyz`，前身是`ghp.ci`，再前身是`ghproxy.com`。选择一个长期可用的镜像为佳。

配置 bucket 镜像：

先看一下当前的源：

```shell
scoop bucket list
```

把之前所有的 bucket 移除了（默认只有 main ）：

```shell
scoop bucket rm main
scoop bucket rm extras
```

然后填写新的镜像 bucket ：

```shell
scoop bucket add main https://ghgo.xyz/https://github.com/ScoopInstaller/Main
scoop bucket add extras https://ghgo.xyz/https://github.com/ScoopInstaller/Extras
```

虽然桶的源改成镜像了，软件包的元信息可以正常获取，但如果软件的下载源来自 github ，可能还是不行。

再来个下载源也是镜像的究极镜像 bucket ，比如：

```shell
scoop bucket add scoop-cn https://ghgo.xyz/https://github.com/duzyn/scoop-cn
```

这种情况下，安装软件（比如 lux ）使用：

```shell
scoop install scoop-cn/lux
```

详见：

- [duzyn/scoop-cn: 中国用户能用的 Scoop 应用库，每日同步 Scoop 的官方库，加速应用的下载速度](https://github.com/duzyn/scoop-cn)
- [lzwme/scoop-proxy-cn: 适合中国大陆的 Scoop buckets 代理镜像库。从多个开源 bucket 仓库同步更新，包含应用 1.6w+。](https://github.com/lzwme/scoop-proxy-cn)

## 附录

清理垃圾：

```shell
# 显示缓存
scoop cache
# 清理下载安装包缓存
scoop cache rm -a
# 清理旧版本
scoop cleanup *
```

查询：

```shell
# 列出已安装的软件
scoop list
# 列出可更新的软件
scoop status
# 查看 scoop 源
scoop config
# 查看 bucket 源
scoop bucket list
```

更新：

```shell
# 更新 scoop 自身
scoop update
# 更新 lux
scoop update lux
# 卸载 lux
scoop uninstall lux
```

帮助：

```shell
scoop help
```
