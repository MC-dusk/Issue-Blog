# Vercel启用Corepack功能

Vercel 默认会根据锁文件（比如pnpm-lock.yaml）自动推断包管理器类型（此处为pnpm）及其版本。但有时候版本会推断错误，而package.json 如果指定了版本，可能会不匹配而编译报错。

比如如果指定pnpm版本为10，而推断使用了9：

"engines": {
  "node": ">=20",
  "npm": ">=11",
  "pnpm": ">=10"
},

那么会报错：

Your pnpm version is incompatible with "/vercel/path0".
Expected version: >=10
Got: 9.15.9
This is happening because the package's manifest has an engines.pnpm field specified.
To fix this issue, install the required pnpm version globally.

为了解决这个问题，需要在项目的环境变量里新建ENABLE_EXPERIMENTAL_COREPACK并赋值为1，来启用 node 的 Corepack 功能，这个功能可以在package.json里面指定需要使用的包管理器及其版本。

{
  "packageManager": "pnpm@10.28.1"
}

  • https://vercel.com/changelog/corepack-experimental-is-now-available (opens new window)
