# 浏览器导出的pdf，部分汉字编码错误问题

https://github.com/BingyanStudio/LapisCV/issues/52

使用思源字体（或其变种），同时使用chromiun系浏览器（或套壳）（比如edge、chrome、vscode、typora）导出pdf，其中部分汉字的unicode码位会错误地从CJK Unified Ideographs区块转为Kangxi Radicals区块。

如果使用火狐浏览器打印pdf，或者手动修改css文件，使用其他字体（比如霞鹜文楷），就正常了。

这些汉字外表看起来和原版很相似，或者就看不出来区别，但一旦复制粘贴，就可能无法识别或乱码。

这些有问题的字都是康熙字典里面的偏旁部首，比如“用”、“户”等等，大概200个字，下面是对比：

错误：

U+2F64 ⽤
U+2F3E ⼾

正确：

U+7528 用
U+6237 户

应该是 PDF 的 ToUnicode 表干的。

字体的 cmap 表存储的是一个 Unicode code point 到 glyph id 的映射，这里的不同 code point 可以映射到同一个 glyph，比如用 fonttools (opens new window) 导出思源黑体的 cmap 表，可见：

<map code="0x2f64" name="uni2F64"/><!-- KANGXI RADICAL USE -->
<map code="0x7528" name="uni2F64"/><!-- CJK UNIFIED IDEOGRAPH-7528 -->

在某些字体中，比如思源黑体，对于 Kangxi Radicals 和 CJK Unified Ideographs 中共有的一些文字使用了相同的字形。

而在 PDF 中，存储的是字形 glyph，以及一个把 glyph 转换到 Unicode code point 的 ToUnicode 表。所以在遇到这种 一个字形对应多个码点 的情况下就会产生冲突，需要选取一个码点。可能是在这个过程中，康熙部首区的码点被优先选中，导致复制时出现错误。
